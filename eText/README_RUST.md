# 南网102-e文件解析工具 - Rust 版本 v2.0.0

## 项目概述

这是使用 Rust 和 Slint GUI 框架重写的南网102-e文件解析工具，保留了所有 Python/PySide6 版本的功能。

## 主要功能

1. **文件解析**
   - 支持 .dat 文件格式
   - 支持 .tar.gz 压缩包格式
   - 多编码支持（UTF-8, GBK, GB2312, GB18030, UTF-16）

2. **GUI 界面**
   - 文件选择功能
   - 进度条显示
   - 状态信息显示
   - 多标签页展示（文件概览、详细数据、原始内容、曲线）

3. **数据展示**
   - 文件概览：显示文件信息和统计数据
   - 详细数据：分段展示数据表内容
   - 原始内容：显示原始文件内容
   - 曲线绘制：支持数据可视化（可选功能）

4. **导出功能**
   - 支持导出解析报告为 .txt 文件

5. **性能优化**
   - 支持限制显示行数以提高性能
   - 多线程解析不阻塞 UI

## 构建和运行

### 前置要求

- Rust 工具链（1.70 或更高版本）
- Windows 10/11

### 构建步骤

```bash
# 1. 克隆或下载项目到本地

# 2. 进入项目目录
cd D:\001\eText

# 3. 构建项目
cargo build --release

# 4. 运行程序
cargo run --release
```

### 直接运行可执行文件

构建完成后，可执行文件位于：
```
target\release\efile_parser_gui.exe
```

## 技术栈

- **GUI 框架**: Slint 1.8
- **解析库**: regex, chrono, encoding_rs
- **压缩处理**: flate2, tar
- **文件 I/O**: std::fs, tempfile
- **错误处理**: anyhow, thiserror
- **序列化**: serde, serde_json

## 与 Python 版本的差异

### 相同功能
- 所有核心功能保持不变
- 界面布局和用户体验保持一致
- 支持的文件格式和编码格式相同

### 改进之处
- **性能**: Rust 版本解析速度更快
- **内存使用**: 更低的内存占用
- **稳定性**: 编译型语言，运行更稳定
- **部署**: 单一可执行文件，无需 Python 环境

### 暂不支持的功能
- 主题切换（Python 版本支持 Windows 注册表读取实现亮/暗主题）
- 完整的曲线绘制（Slint 的绘图功能相对有限）
